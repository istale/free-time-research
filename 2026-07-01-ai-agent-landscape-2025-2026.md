# AI Agent Landscape 2025–2026: A Technical Deep Dive

- 閱讀時間：2026-07-01

## 摘要

2025 年 AI Agent 生態系經歷根本性轉型：所有主要 AI 公司皆已推出 production-ready 的 Agent SDK，MCP（Model Context Protocol）成為通用連接標準，而 **context engineering（上下文工程）** 更取代了 prompt engineering 成為關鍵技術領域。本文覆蓋六大核心面向：

1. **Claude Agent SDK**（Anthropic, 2025/9）：主打「給 Claude 一台電腦」典範，SDK 包含終端、檔案系統、web search 等工具，支援長任務（30+ 小時自主編碼）、自動 context compaction（記憶工具組合後效能提升 39%）。

2. **OpenAI Agents SDK**（2025/3）：最小化抽象設計，以 Agents、Handoffs、Guardrails、Sessions、Tracing 為核心元件。Multi-agent handoff 為一等公民。AgentKit/Agent Builder 視覺化工作流工具被 Sam Altman 形容為「打造 agent 的 Canva」。

3. **Manus AI**：2025 年最大黑馬，8 個月達 $100M ARR 並被 Meta 以 $2–3B 收購。技術核心是「Stochastic Graduate Descent」——重構框架五次以優化 context shaping；KV-cache 優化達 10x 成本削減，檔案系統作為外部記憶實現 100:1 壓縮比。

4. **Context Engineering 四項核心操作**（LangChain 2025/7）：
   - **Write**（寫出窗外）、**Select**（RAG/語意檢索提升 3x 準確率）、**Compress**（auto-compact）、**Isolate**（多 Agent 隔離，提升 90.2% 成功率但代價是 15x token 成本）

5. **Context Engineering 四種失敗模式**：context poisoning、distraction、confusion、clash

6. **Workflow vs Agent 取捨**：Workflow 成本低 4x、可預測；Agent 適合無法預先定義的靈活性任務

## 3W1H 分析
- **What（主題）**: 2025-2026 年 AI Agent 技術全景——各主要公司的 SDK 架構、context engineering 作為核心技術紀律、以及 workflow/agent 兩種典範的適用場景。
- **Why（為什麼重要）**: 這是少見跨六大家（Anthropic、OpenAI、Manus、Meta、Google、LangChain）的系統性技術分析。Context engineering 的四項操作與四種失敗模式，對於建構 production-level agent 系統有直接實務價值。
- **How（如何運作/實作）**: 
  - Claude SDK：雙 Agent 架構（initializer + coding agent）+ MCP 原生整合
  - OpenAI SDK：Handoff 作為第一類原語 + 視覺化 AgentBuilder
  - Manus：外部檔案系統代替內部記憶 + 穩定 KV-cache 前綴達成 10x 成本削減
  - Context 工程：Write/Select/Compress/Isolate 四操作對應不同實現技術
- **Insight（個人心得）**: Context engineering 的崛起最令我印象深刻——過去大家只在意 model 本身的能力，但現在共識是「工程師的核心工作就是 context engineering」。Manus 的「Stochastic Graduate Descent」迭代框架重構的方法論也很值得借鏡，反覆優化 context shaping 而不是一次到位。對於我們這種研究型 Agent 的開發者來說，如何設計有效的 context 管理機制，遠比單純追求更強的 model 更關鍵。
